grep

发表于 2017-07-08 | 分类于 linux | | 阅读次数

主题简介

　　linux中有三个常用的文本处理工具：grep、sed、awk，个人感觉他们就像倚天剑、屠龙刀、玄铁剑一样的江湖利器。是否熟练使用这三个工具，是检验你linux功力的试金石。因为内容比较多，我用三篇文章来分别讲述这三个工具的常见使用方式。这篇文章首先来讲述grep命令。

GREP命令

Grep命令的由来

　　grep这个名字的由来可以追溯到ex编辑器。如果启动ex编辑器来查找某个字符串，需要在ex的命令提示符后键入：
　　:/pattern/p
　　字符”p”表示打印命令，所以包含字符串pattern的第一行的内容会被打印出来。如果希望打印所有包含pattern的行，可以输入：
　　:/g/pattern/p
　　当g出现在pattern前面时，其含义是“文件中打所有行”，或“执行全局替换”。
　　被查找的模式称作正则表达私（regular expression）,因此我们可以用RE来替换pattern,于是上面这条命令就变成了：
　　:g/RE/p
　　说到这，您就应该清楚grep的含义和它的名称的来源了。它表示“全局查找正则表达式并且打印结果行。”使用grep的好处在于，不需要启动编辑器，就可以执行查找操作，也不用把正则表达式括在正斜杠里。使用grep比不使用ex或vi快的多。

Grep命令格式

　　grep命令在一个或多个文件中查找某个字符串模式。如果这个模式中包含空格，就必须用引号把它括起来。在这里我的建议是，使用grep命令时，都把模式用单引号或双引号括起来，不然可能会产生不想要的效果。后面我们会举例。位于模式之后的所有单词都被视为文件名。grep将输出发送到屏幕，它不会对输入文件进行任何修改或变化。
　　grep word file1 file2
　　grep程序的输入可以来自管道，而不仅仅是文件。例如：
　　ps -ef | grep root
　　什么是管道，后面会有文章单独介绍，这里不再单独介绍。下面通过样例来观察grep命令的用法。grep命令中的正则表达式，需要朋友们自己学习了。想要用好grep、sed、awk命令，正则表达式的熟练使用是必不可少的，需要朋友们多多练习！
