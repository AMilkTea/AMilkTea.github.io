ubuntu16.04下安装oracle 11g问题杂记

发表于 2017-12-03 | 分类于 oracle | | 阅读次数

主题简介

　　ubuntu下安装oracle是个比较麻烦的事，因为oracle本身并不支持ubuntu安装。虽然麻烦，但办法还是有的。我在博客１和博客２这两篇博客的指导下完成了安装。因为博客１对于安装完成后的配置部分并未讲解，因此博客２主要是起补充作用。前面的部分可以完全按照博客１的步骤操作即可。等到出现安装界面后，就可以采用博客２的步骤进行操作。另外，安装完成，如果想要开机自启动oracle，博客１也有相关介绍。但是，除了在博客１介绍的有关问题之外，我在配置开机自启动时也遇到了一些其他问题。下面主要分享下这些问题以及他们的解决办法。

问题

问题１　oracle开机启动失败

　　如果开机自启动的配置是按照博客１中的描述来进行配置的，可以查看日志/var/log/oracle，看看日志中是否有报错。如下图所示：
　
　　发现，日志中并没有什么错误信息。但是，数据库启动的日志并没有打出来，因此，可以断定脚本肯定是有错误，导致，数据库启动程序并没有被调起。手动执行相关shell脚本，结果如下：
　
　　果然不出所料，脚本没有找到数据库启动脚本，导致此问题最大的可能性便是路径配置问题。仔细检查，发现/etc/init.d/oracle脚本中的命令su $ORA_OWNER -c “$ORACLE_HOME/bin/lsnrctl start” 写成了su $ORA_OWNER -c “$ORA_HOME/bin/lsnrctl start” 。因为我当初配置开机自启动查询了多篇博客，可能复制粘贴时变量不一致导致的。将全文中这个地方的错误都改正后再次手动执行脚本，结果如下：

　　发现数据库能正常启动了，但还报了两处错误。

问题２　/etc/init.d/oracle: 行 17: -d: 未找到命令　

　　仔细观察相关行，发现!写成了｜，应该是抄写脚本的时候将!看成了｜,导致错误。不过这个错误并没有影响整个流程。这一行脚本的作用是判断文件与路径是否存在。

问题３　/dbhome_1/bin/emctl: 23: ulimit: bad number

这个错误比较奇怪，查询后发现可以把”$ORACLE_HOME/bin/emctl”脚本的’#!/bin/sh’改为’#!/bin/bash’即可修复问题。

总结

　　发现问题，首先要定位问题，定位问题的一般方法是查找日志，分析错误。问题定位后，就可以找对症下药了。所以，面对问题时，不要盲目寻医，一定要找准症结在什么地方后，再找对应的解决方案。三个问题解决后，就可以正常启动oracle了。如下图所示：
